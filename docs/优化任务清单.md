# md2ui 优化任务清单

> 对标 VitePress / GitHub / GitBook / Notion，结合社区真实痛点，梳理 md2ui 的功能演进路线。

## 当前优势（保持并强化）

- 零配置启动：一行命令 `md2ui` 即可预览，对比 VitePress 需要 init + 配置 sidebar/nav
- Mermaid 开箱即用：VitePress 需要额外安装插件配置
- 图片/Mermaid 点击放大：主流方案都没有内置
- 三栏可拖拽布局：比 VitePress 固定宽度更灵活
- 站内链接自动解析：.md 互链自动改写为 SPA 路由

---

## P0 - 核心体验（必须做）

### 1. 全文搜索

现状：完全没有搜索能力，文档一多只能靠目录树翻找。

社区痛点：
- GitBook 免费版搜索受限，付费版 $79/月起步，很多团队因此寻找替代方案
- VitePress 的 local search 对中文分词支持一般
- 开发者调研显示 93% 的人认为"找不到内容"是使用文档的最大障碍

方案建议：
- 前端侧全文搜索，使用 flexsearch 或 minisearch
- 加载文档列表时建立索引，支持中文分词
- 快捷键 Ctrl/Cmd + K 唤起搜索面板
- 搜索结果高亮匹配文本，点击直接跳转

### 2. 代码语法高亮

现状：代码块只有深色背景 + 白色文字，没有 token 级着色。

对比：
- VitePress 用 Shiki（VS Code 同款引擎），支持行高亮、行号、diff
- GitHub 有完整的语法高亮

方案建议：
- 接入 highlight.js（轻量）或 Shiki（效果好但体积大）
- 推荐先用 highlight.js，后续可切换
- 同时加上"复制代码"按钮（代码块右上角）

### 3. 暗色模式

现状：只有亮色主题，颜色硬编码在 CSS 中。

社区痛点：
- 暗色模式已经是开发者工具的标配
- 长时间阅读文档，亮色主题对眼睛不友好

方案建议：
- 用 CSS 变量重构整套颜色体系
- 支持三种模式：亮色 / 暗色 / 跟随系统
- 顶部或侧边栏加切换按钮
- localStorage 持久化用户偏好

## P1 - 体验提升（优先做）

### 4. 移动端适配

现状：
- 768px 以下侧边栏直接挤压内容区，没有抽屉模式
- 没有汉堡菜单
- TOC 在 1200px 以下直接隐藏，无替代方案

方案建议：
- 侧边栏改为 overlay 抽屉模式（左滑出）
- 加汉堡菜单按钮
- TOC 改为底部弹出面板或浮动按钮触发
- 触摸手势支持（左右滑动切换文档）

### 5. 上一篇/下一篇导航

现状：阅读完一篇文档后，只能回到侧边栏点击下一篇。

对比：VitePress / GitBook / Notion 都有文档底部的前后导航。

方案建议：
- 文档底部添加"上一篇"/"下一篇"按钮
- 按目录树的顺序自动计算前后关系
- 键盘快捷键支持（左右方向键）

### 6. 复制代码按钮

现状：代码块没有复制按钮，用户需要手动选中复制。

对比：VitePress / GitHub / GitBook 都有一键复制。

方案建议：
- 代码块右上角添加复制图标
- 点击后显示"已复制"反馈
- 可以和语法高亮一起做

### 7. 文档元信息展示

现状：没有显示文档的最后更新时间、字数统计等信息。

社区痛点：
- 开发者经常抱怨文档过时但无法判断更新时间
- 93% 的开发者认为过时文档是使用开源项目的最大障碍

方案建议：
- CLI 模式下通过 fs.stat 获取文件修改时间
- 文档顶部或底部显示"最后更新于 xxx"
- 可选：显示预估阅读时间（按字数计算）

## P2 - 差异化功能（有竞争力）

### 8. Frontmatter 支持

现状：不支持 YAML frontmatter 元数据。

对比：VitePress / GitBook 都支持 frontmatter 配置页面标题、描述、布局等。

方案建议：
- 解析 markdown 文件头部的 YAML frontmatter
- 支持 title / description / order / hidden 等字段
- 用 frontmatter 的 title 覆盖文件名作为导航标签
- hidden: true 可以隐藏文档不在导航中显示

### 9. 导航栏搜索过滤

现状：侧边栏只有展开/收起，文档多了不好找。

社区痛点：VitePress 的 sidebar 配置是手动维护的，文档多了配置文件巨长。md2ui 自动生成目录是优势，但缺少快速过滤。

方案建议：
- 侧边栏顶部加一个过滤输入框
- 实时过滤匹配的文档名
- 和全文搜索互补：过滤框找文档名，搜索面板找内容

### 10. 自定义配置文件

现状：只有 config.js 里的 defaultPort 和 folderExpanded 两个配置项。

社区痛点：
- VitePress 配置过于复杂（sidebar/nav/theme 配置动辄几百行）
- GitBook 的 SaaS 模式不支持自托管，定制能力受限

方案建议（保持轻量原则）：
- 支持项目根目录下的 md2ui.config.js 或 .md2uirc.json
- 可配置项：站点标题、Logo、主题色、GitHub 链接、页脚内容
- 保持零配置可用，配置文件纯可选

### 11. 文档热更新

现状：CLI 模式下修改 .md 文件后需要手动刷新页面。

方案建议：
- 利用 Vite 的 HMR 能力，监听 .md 文件变化`
- 文件修改后自动推送更新到浏览器
- 新增/删除文件时自动更新导航树

## P3 - 进阶能力（锦上添花）

### 12. SSG 静态构建

现状：纯运行时渲染，部署必须跑 Node 服务，SEO 为零。

对比：VitePress / GitBook 都能生成纯静态 HTML。

方案建议：
- 新增 `md2ui build` 命令
- 预渲染所有文档为静态 HTML
- 产物可直接部署到 CDN / GitHub Pages / Vercel
- 这是从"预览工具"升级为"文档站生成器"的关键一步






### 16. PWA 离线访问

方案建议：
- 注册 Service Worker 缓存文档内容
- 支持离线阅读已访问过的文档
- 添加"安装到桌面"提示
